# コードレビュー手順

1. `src/` ディレクトリ内の全ソースファイルを読み取る
2. 以下の観点でレビューする:
   - バグの可能性がある箇所
   - セキュリティ上の問題（入力バリデーション、インジェクション等）
   - パフォーマンスへの影響
   - エラーハンドリングの不足
   - コードの可読性・保守性
3. 指摘事項を重要度順（Critical > Warning > Info）にまとめる
4. 各指摘に具体的な修正案を含める
